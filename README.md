## 📚 단계별 실습 파이프라인 (Exercise Pipeline)

이 실습은 4개의 주피터 노트북(Jupyter Notebook) 파일로 구성되어 있습니다. 
아래 순서대로 파일을 열고 각 셀(Cell)을 차례대로 실행하며 주석과 코드를 학습해 보세요.

### 1️⃣ `1_Data_Preprocessing.ipynb` (데이터 전처리)
- **학습 내용:** 제공된 `Dr.tail` 원시 환자 데이터셋(`sample_data.csv`)에서 증상, 반려동물의 종과 나이, 보호자가 입력한 메모 등을 읽어들입니다.
- **주요 코드:** Pandas를 이용해 데이터를 파싱하고, 파이썬의 f-string을 활용하여 여러 파편화된 정보를 **'하나의 자연스럽고 통합된 긴급 진단 프롬프트(질문)'** 텍스트로 가공(Formatting)하는 방법을 배웁니다.

### 2️⃣ `2_Single_Agent.ipynb` (단일 에이전트 구축)
- **학습 내용:** CrewAI 프레임워크의 가장 기본적인 단위인 **에이전트(Agent)**와 **임무(Task)**의 개념을 배웁니다.
- **주요 코드:** LLM(GPT-4o)을 활용하여 "종합 수의학 컨설턴트"라는 페르소나(역할)를 부여받은 단 하나의 뛰어난 에이전트를 생성합니다. 이 에이전트 혼자서 전처리된 보호자의 질문을 읽고, 긴급도를 판단하며, 예상 질병군을 추론하고, 최종 조언을 텍스트로 작성하는 전체 과정을 단독으로 수행하게 됩니다.

### 3️⃣ `3_Multi_Agent_Crew.ipynb` (멀티 에이전트 시스템 협업)
- **학습 내용:** 단일 에이전트의 한계를 극복하기 위해, 역할을 잘게 쪼개어 여러 에이전트가 소통하는 **다중 에이전트(Multi-Agent) 크루**를 설계합니다.
- **주요 코드:** 이 노트북에서는 3명의 전문가를 고용합니다.
  1. **분류 간호사(Triage):** 먼저 질문을 읽고 긴급도(낮음~응급)를 빠르게 판단합니다.
  2. **수의사(Vet):** 간호사의 긴급도 판단 결과(Context)를 넘겨받아 의학적 진단을 내립니다.
  3. **상담가(Advisor):** 수의사의 진단 결과와 환자의 질문을 모두 종합(Context)하여 보호자에게 알기 쉬운 언어로 최종 리포트를 작성합니다.
  결과물이 단일 에이전트와 비교해 어떻게 차이가 나는지 집중적으로 관찰해 보세요.

### 4️⃣ `4_Evaluation.ipynb` (자동 채점 및 시각화 비교)
- **학습 내용:** 최신 인공지능 성능 평가 트렌드인 **LLM 심사위원(LLM-as-a-Judge)** 기법을 활용하여 내가 만든 시스템의 성능을 객관화합니다.
- **주요 코드:** GPT-4o에게 "당신은 수의학 전문 평가자입니다"라는 역할을 주고, **실제로 현업 수의사가 작성했던 정답(Ground Truth)** 텍스트와 **AI가 작성한 답변**을 나란히 보여주며 비교 채점을 지시합니다.
- 정확도(Accuracy), 완전성(Completeness), 공감능력(Empathy)이라는 3가지 지표를 분석해 JSON 점수로 반환받은 뒤, 최종적으로 `Matplotlib`을 활용해 Single Agent vs Multi Agent vs 실제 수의사의 성능을 막대/선 그래프로 시각화(Visualization)하여 한눈에 비교합니다.
